
# Founder Narrative – Allocator Decision Engine

Investment analysts in fund-of-funds platforms do not lack intelligence.
They lack structured workflow infrastructure.

Every month, analysts receive performance data from external managers.
The data arrives inconsistent, incomplete, and misaligned.
Before any investment insight can be formed, the analyst must:

- Normalize formats
- Align time series
- Detect missing data
- Validate anomalies
- Reconcile definitions

This preprocessing step is repetitive and fragile.
It lives in spreadsheets that are difficult to audit and nearly impossible to reproduce cleanly.

After normalization, metrics are computed.
Memos are drafted.
Claims are written.

But there is no structured link between:

Raw data → Metrics → Investment claims.

The Allocator Decision Engine solves this gap.

It enforces:

1. Canonical normalization of messy manager data.
2. Deterministic, reproducible metric computation.
3. Constraint-driven ranking aligned with mandate.
4. Structured memo generation grounded in precomputed facts.
5. Claim-to-data traceability for committee defense.

This is not an AI robo allocator.
This is not a dashboard.
This is not a DDQ tool.

It is a decision kernel for alternative investment analysts.

Version 1 focuses on efficiency and trust.
Later versions evolve into institutional memory, portfolio simulation, and cross-manager intelligence.

But it begins with something simple and powerful:

Make the analyst’s workflow structured, repeatable, and defensible.
